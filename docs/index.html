How It Works

Authentication pipeline

Every step runs on the device. No network call. No server. No compromise.

01
Camera Capture + EXIF Correction
React Native Vision Camera captures a real-time frame. EXIF orientation is corrected before any processing — a common failure point on Android that we explicitly handle.
vision-camera v4
02
Face Detection + Quality Gate
MediaPipe detects the face and runs quality checks — brightness, sharpness, face size, coverage, yaw/roll angles. Blurry, dark, extreme-angle, or partial faces are rejected before inference.
MediaPipe · Kotlin
03
Alignment + Normalization
Face is cropped, aligned, and resized to 112×112. Pixels are normalized to [−1, 1]. CLAHE preprocessing compensates for outdoor lighting conditions.
OpenCV · Bitmap
04
MobileFaceNet Inference (TFLite)
The preprocessed 112×112 face is passed to MobileFaceNet running via TFLite inside a Kotlin native module. Output is a 128-dimensional embedding vector.
TensorFlow Lite · MobileFaceNet
05
L2 Normalization + Cosine Matching
Embedding is L2-normalized. Cosine similarity (dot product) is computed against all enrolled users. Authentication requires best score ≥ 0.72 AND a margin of ≥ 0.04 over the second-best match.
Cosine similarity · Margin rule
06
Offline Attendance Logging
Attendance record (userId, timestamp, score, status) is stored locally. When connectivity is restored, records sync to the server and are purged from local storage after confirmation.
AsyncStorage · NetInfo · AWS sync
Features

Built for the real world

Every design decision was made for field reliability, not lab conditions.

🧬
3-Image Enrollment
Registration captures three poses (frontal, left, right) and averages the embeddings. This gives a more robust, pose-invariant identity representation than single-shot enrollment.
🛡️
Liveness Detection
EAR (Eye Aspect Ratio) blink detection and head pose estimation prevent spoofing via printed photos, phone screens, or static images.
🚫
Duplicate Prevention
Before registering a new user, the system checks all existing embeddings. If similarity ≥ 0.70 with any enrolled user, registration is blocked — preventing the same face appearing under two names.
🔍
Face Quality Rejection
Poor captures are rejected before inference: blurry, overexposed, underexposed, partially out of frame, too small, too close, extreme head angles. Garbage in = garbage embedding.
📴
100% Offline
TFLite model, embeddings, and attendance logs live entirely on-device. No network call is made during authentication. Works in tunnels, forests, remote highways — anywhere.
🔄
Smart Sync + Purge
NetInfo monitors connectivity. When online, queued attendance records are batch-uploaded. Records are purged from local storage only after server acknowledgement — zero data loss.
